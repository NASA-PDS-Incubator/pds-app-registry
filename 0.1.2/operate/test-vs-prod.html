Test vs Production

This section describes few Test vs Production use cases

Use a test environment to play around with registry data

Install Solr, Registry Manager and Harvest on your laptop as described in Installation section.

You can use multiple collections at the same time. It doesn't have to be default 'registry' collection.

The following example will create 3 collections, test1, test2, and test3. 

registry-manager create-registry -collection test1
registry-manager create-registry -collection test2
registry-manager create-registry -collection test3

You can customize individual collections as described in Registry Customization

For example, to create collection test1 from customized Solr config set stored in /tmp/test1 directory, call this command 

registry-manager create-registry -configDir /tmp/test1 -collection test1

You can easily delete any of your test collections and start over. For example, 

registry-manager delete-registry -collection test1

Swap all test data into operations

Extract all data from a test collection into an XML file, as shown below 

registry-manager export-data -file /tmp/mydata.xml -all -collection test1

Then load the XML file into production environment. In the following example 3-node ZooKeeper cluster is used to load data into multi-node production cluster. 

registry-manager load-data -file /tmp/mydata.xml -zkHost zk1.prod.local:2181,zk2.prod.local:2181,zk3.prod.local:2181/solr

Instead of ZooKeeper connection string you can also use a URL of a Solr node. 

registry-manager load-data -file /tmp/mydata.xml -solrUrl http://my.prod.local:8983/solr

NOTE: You may need to copy (FTP) your XML file into production environment.

Swap a subset of test data into operations

Extract a subset of data by packageId from a test collection into an XML file, as shown below 

registry-manager export-data -file /tmp/mydata.xml -packageId 8d8ae96d-044e-473d-a278-62635b1c5977 -collection test1

Then load the XML file into production environment. 

registry-manager load-data -file /tmp/mydata.xml -solrUrl http://my.prod.local:8983/solr

Rollback changes made in operations

Each time you run Harvest tool, it generates unique packageId (UUID) and stores it in each Solr document ("_package_id" field). If you loaded some data and want to delete it, find out the packageId from Harvest generated file. Then run the following command to delete all documents by that packageId, 

registry-manager delete-data -packageId 8d8ae96d-044e-473d-a278-62635b1c5977 -solrUrl http://my.prod.local:8983/solr

You can also delete documents by lidvid or lid. 

registry-manager delete-data -lidvid urn:nasa:pds:context:target:asteroid.4_vesta::1.1

NOTE: You can overwrite existing documents by loading new documents with the same lidvids. Lidvid is a primary key in the registry. In this case, there is no way to restore original documents.
